# 🧋 果茶 APP 部署指南

## 文件清单

| 文件 | 用途 |
|------|------|
| `index.html` | 前端页面（单文件，包含所有代码和图片） |
| `apps-script.js` | Google Apps Script 后端代码 |

---

## 第一步：部署 Google Sheets 后端

这一步让你的配方数据保存在云端（Google Sheets），不只存在浏览器里。

### 1. 新建 Google Sheet

打开 [sheets.google.com](https://sheets.google.com)，新建一个空白表格，随便起个名（比如"果茶数据"）。

### 2. 创建两个 Sheet Tab

在表格底部，点 `+` 新建 sheet tab，分别命名为：
- `recipes`
- `ingredients`

⚠️ 名字必须完全一致（小写英文），否则代码找不到。

### 3. 粘贴 Apps Script 代码

1. 点击菜单 **Extensions → Apps Script**
2. 删掉默认的 `function myFunction(){}`
3. 把 `apps-script.js` 里的全部代码粘贴进去
4. 点左上角保存（Ctrl+S）

### 4. 部署为 Web App

1. 点击右上角 **Deploy → New deployment**
2. 左侧选 **Web app**
3. 设置：
   - **Execute as**: `Me`（用你自己的 Google 账号运行）
   - **Who has access**: `Anyone`（允许任何人访问，这样你的网页才能调用）
4. 点 **Deploy**
5. 第一次会要求授权，点 **Authorize access** → 选你的 Google 账号 → 点 **Allow**
6. 复制弹出的 **URL**（长这样：`https://script.google.com/macros/s/一长串ID/exec`）

### 5. 把 URL 填进 index.html

打开 `index.html`，搜索这一行：

```
const SCRIPT_URL = 'https://script.google.com/macros/s/你的部署ID/exec';
```

把 URL 替换成你刚复制的那个。保存。

## 第二步：部署到 GitHub Pages

### 如果你已经有 GitHub 仓库

1. 把 `index.html` 推到仓库（放在根目录）
2. 去仓库 **Settings → Pages**
3. Source 选 `Deploy from a branch`，选 `main` 分支，`/ (root)`
4. 保存，等 1-2 分钟
5. 页面上会出现你的网址：`https://你的用户名.github.io/仓库名/`

### 如果你还没有仓库

1. 去 [github.com/new](https://github.com/new) 新建仓库（比如叫 `bubble-tea`）
2. 勾选 **Add a README file**
3. 创建后，点 **Add file → Upload files**
4. 把 `index.html` 拖进去，commit
5. 然后按上面的步骤开 Pages

---

## 第三步：添加到手机桌面（PWA 效果）

### iPhone
1. 用 Safari 打开你的 GitHub Pages 网址
2. 点底部分享按钮 → **添加到主屏幕**

### Android
1. 用 Chrome 打开网址
2. 点右上角菜单 → **添加到主屏幕**

---

## 离线说明

- 没有网络时，App 用浏览器里的 localStorage 缓存数据
- 恢复网络后会自动同步到 Google Sheets
- 如果 Apps Script URL 没填或填错了，只会报一个 console 警告，**不影响本地使用**

---

## 更新 Apps Script 后怎么办

如果改了 `apps-script.js`：
1. 去 Apps Script 编辑器粘贴新代码
2. 点 **Deploy → Manage deployments**
3. 点编辑图标（铅笔），版本选 **New version**
4. 点 **Deploy**

⚠️ URL 不变，不用改 `index.html` 里的地址。

---

## 排错

| 症状 | 原因 | 解决 |
|------|------|------|
| Console 报 `Failed to fetch` | SCRIPT_URL 没填 / 填错 | 检查 URL 是否正确 |
| 数据不同步 | Apps Script 权限问题 | 重新 Deploy，确保 Who has access = Anyone |
| 页面空白 | 浏览器太旧 / JS 报错 | 用 Chrome/Safari 最新版，开 console 看错误 |
| 图片不显示 | base64 被截断 | 确认 index.html 文件完整（~342KB） |
